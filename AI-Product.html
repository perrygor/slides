■INSIGHT · F-04
SPEED
F-04 Speed

速度是第一性原理

"Ship it on Friday, learn from it on Monday. The cost of delay always exceeds the cost of imperfection."

— Alex Albert, Anthropic
「周五上线，周一复盘。延迟交付的代价永远超过不完美的代价。」
  • ■完美是交付的敌人，速度本身就是竞争力
  • ■每一天的延迟都是信息损失
  • ■先上线，再迭代——反馈回路越短越好
座舱映射 · OTA 节奏
Anthropic 节奏74 features / 52 days ≈ 1.4/day
Cycle月 → 周 → 天（Cat Wu）
PrincipleShip → Measure → Learn

■INSIGHT · F-05
SPEC
F-05 Spec

极简 Spec

"For specs, we write like 10 bullets and that's it"

— Alex, Codex Team
「写规格说明，我们大概写 10 条要点就够了。」
  • ■复杂文档 = 延迟，简单意图 = 生产力
  • ■PRD 没消失，变成 one-pager：目标 + 核心场景 + failure case
  • ■"写最少的文档，让最多的人能行动"
座舱映射 · 需求文档 ≤ 10 bullets

■INSIGHT · F-06
PLAN
F-06 Plan

不做中期规划

"Plan short and long term, never medium term"

— Codex Team
「只做短期和长期规划，永远不做中期规划。」
  • ■短期（8周）：具体里程碑
  • ■长期（1年+）：模型能力演进方向
  • ■中期（6-12月）直接跳过 — AI 进化太快
座舱映射 · 删除半年规划，只保留 6 周目标 + 1 年愿景

■WORKMODE · F-10
NATIVE
F-10 AI-Native

你的工作方式
决定了你能构建什么

"90% of Claude's code is now written by AI."

— Mike Krieger, Anthropic CPO
「Claude 90% 的代码现在都由 AI 自己编写。」

停留在 Copilot 模式的团队
只能做出 Copilot 级别的产品。
工作方式本身，就是产品品味的体现。

■ 阶段 1 · 现在
辅助 Copilot
人写，AI 补。AI 是加速器，人做所有决定
■ 阶段 2 · 下一步
协作 Pair
人和 AI 并肩，各有分工。定义人该做什么、AI 该做什么
■ 阶段 3 · 终局
编排 Orchestrate
人指挥，多个 Agent 执行。判断力 = 定义目标 + 编排策略
座舱映射 · PM 用 AI 写 PRD → PM 和 AI 一起定义需求 → PM 指挥多个 Agent 并行出方案

从 Process 到涌现

"要构建还不完全能用的产品"

— Cat Wu

Judgment
核心目标 + 底线约束
提升体感、禁止直吹…

■CASE STUDY · F-13
CASE
F-13 Case

用户说"有点冷"

古典做法（Process）
意图识别 → 分类"温度调节"
→ 触发预设脚本
→ 指令：主驾空调调高 2 C，风速 3 挡
Agent Native 做法
Context：车外 5℃，车窗微开，电池续航仅剩 15%
Judgment：目标"提升体感温度"，约束"克制使用高能耗硬件"
Action：Agent 自主决定 → 关上车窗 + 开启 1 挡座椅加热
